## 1. 개요

### 1.1 AlphaLens 플랫폼 소개

AlphaLens는 캐피탈사(투자법인)를 위한 **한국 주식 AI 멀티팩터 스코어링 플랫폼**이다. 중/장기적 투자 수익 실현을 목표로 하며, 정확한 매수/매도 타이밍 포착이 핵심 KPI이다.

**7-팩터 모델 (현재 가중치)**:

| 팩터 | 가중치 | 분석 방식 | 핵심 지표 |
|------|--------|----------|----------|
| Technical | 23% | 규칙 기반 | MA(5/20/60/120/200), RSI, MACD, OBV, BB, Volume, ATR |
| Fundamental | 19% | 규칙 기반 | PER(섹터별 13개 기준), PBR, ROE |
| Signal | 19% | 규칙 기반 | Momentum, Mean Reversion, Breakout, Sell; 레짐 감지 |
| Macro | 14% | 규칙 기반 | US시장, FX, 금리, 원자재, 중국; yfinance 데이터 |
| Risk | 15% | 규칙 기반 | 변동성(40%), MDD(35%), 유동성(25%); A-E 등급 |
| Related | 5% | 규칙 기반 | 관련기업 모멘텀, 민감도 x0.5, +-3 cap |
| News | 5% | **AI 기반** | KR-FinBERT 앙상블 (Sprint 4에서 활성화) |

**기술 스택**: FastAPI + Vanilla JS(Chart.js) + Docker Compose, 비동기 httpx, WebSocket 실시간 스트리밍

### 1.2 AI/LLM 도입 철학

> **"정량은 규칙, 정성은 LLM"**

- 기술적 분석(MA, RSI, MACD), 시그널 로직(모멘텀, 평균 회귀), 리스크 계산(VaR, MDD) 등 **정량 지표는 수학적 규칙으로 처리**한다
- 뉴스 감성 분석, 지정학 이벤트 해석, 애널리스트 리포트 종합 등 **정성적 판단이 필요한 영역에 AI/LLM을 투입**한다
- LLM은 정량 모델을 **"대체"가 아닌 "보강"**하는 역할로 한정한다

### 1.3 현재 상태 (Sprint 1-4 기준)

**완료된 주요 작업**:
- Sprint 1: 시스템 안정성 확보, 치명적 모델 버그 수정 (Risk veto, 등급 fallthrough, 매수/매도 균형)
- Sprint 2: 스코어링 정확도 향상 (다중공선성 해소, ADX, 매도 시그널 강화, VaR/CVaR)
- Sprint 3: 데이터 인프라 확장 (매크로 다변화, 지정학 9개 카테고리, 프론트엔드 고도화)
- Sprint 4 (진행 중): KR-FinBERT 감성분석 통합, 백테스팅 프레임워크, AI/LLM 로드맵

**현재 AI 활용 현황**:
- `sentiment.py`: 키워드 기반 감성분석 + KR-FinBERT 앙상블 (0.3:0.7) 구현 완료
- `finbert_service.py`: snunlp/KR-FinBert-SC 모델 싱글턴 서비스, LRU 캐시(1000건), lazy loading
- `geopolitical_service.py`: 키워드 기반 9개 카테고리 이벤트 감지, 한영 교차 확인, 섹터 영향 매트릭스

## 2. Phase 1: KR-FinBERT 감성분석 (Sprint 4 - 진행 중)

### 2.1 구현 내용

**snunlp/KR-FinBert-SC 모델 통합**:
- `backend/services/finbert_service.py`: FinBERTService 싱글턴 클래스
  - Lazy model loading (첫 호출 시 모델 로드)
  - LRU 캐시 1,000건 (MD5 해시 키)
  - torch/transformers 미설치 시 graceful fallback
  - 배치 분석 지원 (`analyze_batch`)

**뉴스 팩터 활성화**:
- 기존: News 가중치 0% (참고용) → **변경: News 5% (W_NEWS = 0.05)**
- Technical 25%→23%, Fundamental 20%→19%, Signal 20%→19%, Macro 15%→14% 조정
- Risk 10%→15% 상향 (별도 이슈에 따른 조정)

**키워드 + FinBERT 앙상블**:
- `sentiment.py`의 `analyze_sentiment()` 함수에서 통합
- FinBERT 가용 시: **FinBERT 70% + 키워드 30%** 앙상블
- FinBERT 미가용 시: 기존 키워드 방식 100% (무중단 fallback)
- 제목 기반 FinBERT 분석 (제목이 감성 판단에 가장 중요)

**캐시 기반 성능 최적화**:
- FinBERT 결과 MD5 해시 캐시 → 동일 제목 재분석 방지
- 스코어링 전체 캐시 TTL 300초 (기존 유지)

### 2.2 기대 효과

| 항목 | 키워드 방식 | FinBERT 앙상블 | 개선폭 |
|------|-----------|--------------|-------|
| 감성 분류 정확도 | ~60% | ~85% | +25%p |
| 문맥 의존적 해석 | 불가 | 가능 | 신규 |
| 신조어 대응 | 수동 추가 | 자동 학습 | 자동화 |
| 부정어 처리 | 규칙 기반 | 모델 내재 | 정확도 향상 |

- **문맥 의존적 감성 인식**: "금리 인상" → 은행주에 긍정, 성장주에 부정 (키워드로 구분 불가)
- **신조어/신규 키워드 자동 대응**: 학습된 언어 모델이 새로운 표현도 감성 추론 가능
- **투자 전문 용어 이해**: FinBERT가 금융 도메인 사전학습으로 일반 BERT 대비 높은 정확도

### 2.3 측정 지표

- **감성 분류 F1 Score**: 수동 라벨링 100건 기준 키워드 vs FinBERT 비교
- **뉴스 팩터의 수익률 기여도**: 백테스트에서 News 5% 포함/미포함 수익률 차이
- **앙상블 일관성**: 키워드와 FinBERT의 판단 불일치율 모니터링
- **처리 성능**: FinBERT 평균 추론 시간 (목표 <100ms/건, CPU 기준)

## 3. Phase 2a: 지정학 LLM 보강 (예상 2-3개월)

### 3.1 목표

현재 `geopolitical_service.py`는 **키워드 기반 이벤트 감지**에 의존한다:
- 9개 카테고리(전쟁, 한반도, 관세, 통화정책, 에너지, 중국경제, 반도체, 환율, 공급망)의 250+개 키워드 매칭
- 한영 RSS 교차 확인으로 신뢰도 보정
- 섹터 영향 매트릭스(EVENT_SECTOR_IMPACT)로 자동 매핑

**한계**:
- 키워드 매칭으로는 이벤트의 **심층 맥락**을 파악할 수 없음
- "트럼프 관세 발표" vs "트럼프 관세 철회" → 동일 키워드 매칭, 반대 의미
- 새로운 유형의 이벤트(예: AI 규제, 기후 정책) 대응에 키워드 추가 필요
- 이벤트 간 **연쇄 효과**(2차, 3차 영향) 분석 불가

**Phase 2a 목표**:
- 기존 키워드 탐지를 유지하면서 LLM으로 **심층 분석 계층**을 추가
- 이벤트의 시장 영향 방향/크기를 LLM이 예측
- 시나리오 시뮬레이션 자동화

### 3.2 아키텍처

```
뉴스 수집 (기존 RSS)
    │
    ▼
이벤트 탐지 (기존 키워드 매칭) ──── 1차 필터링
    │
    ▼
LLM 심층 분석 (신규)
    ├── 이벤트 분류 정확도 검증
    ├── 심층 맥락 분석 (방향성, 강도, 시간축)
    ├── 섹터 영향 예측 (기존 매트릭스 보정)
    └── 시나리오 시뮬레이션
    │
    ▼
영향도 평가 → 스코어링 반영
```

**설계 원칙**:
- LLM은 기존 키워드 탐지의 **보조 계층**으로 동작 (대체가 아닌 보강)
- 키워드 탐지에서 감지된 이벤트만 LLM에 전달 (비용 최적화)
- LLM 실패 시 기존 키워드 결과로 fallback (가용성 보장)

### 3.3 후보 모델 비교

| 모델 | 월 비용 추정 | 정확도 | 지연시간 | 한국어 | 권고 |
|------|------------|--------|---------|--------|------|
| GPT-4o | ~$200-500 | 높음 | 1-3초 | 우수 | Phase 2a 초기 추천 |
| Claude Sonnet | ~$150-400 | 높음 | 1-2초 | 우수 | 대안 (구조화 출력 강점) |
| GPT-4o-mini | ~$30-80 | 중 | 0.5-1초 | 양호 | 비용 최적화 시 |
| Llama 3 70B (로컬) | GPU 서버 비용 | 중-높 | 0.5-1초 | 보통 | Phase 2a 후기 (비용 절감) |
| Gemini 1.5 Flash | ~$20-60 | 중 | 0.3-0.8초 | 양호 | 대량 처리 시 |

**권고**: GPT-4o로 초기 구현 → 프롬프트 안정화 후 GPT-4o-mini 또는 로컬 모델로 비용 최적화

### 3.4 구현 계획

**Step 1: 프롬프트 엔지니어링 (2주)**
- 지정학 이벤트 분류 프롬프트 설계
  - 입력: 키워드 탐지 결과 + 관련 뉴스 제목/본문
  - 출력: 구조화된 JSON (이벤트 유형, 방향성, 강도, 영향 섹터, 시간축)
- 시장 영향 분석 프롬프트 설계
  - 단기(1-3일), 중기(1-4주), 장기(1-6개월) 영향 예측
- Few-shot 예시 구축 (카테고리별 5-10개)

**Step 2: 이중 검증 시스템 (2주)**
- 기존 키워드 탐지 결과와 LLM 분석 결과 교차 검증
- 불일치 시 로깅 및 알림 (모델 개선 데이터 축적)
- `geopolitical_service.py`의 `_detect_events()` 결과를 LLM 입력으로 전달

**Step 3: 결과 캐싱 + 배치 처리 (1주)**
- 실시간 아닌 **주기적 갱신** (15분~1시간 주기)
- LLM 분석 결과 캐시 TTL: 기본 30분, 위기 시 10분
- 배치 처리로 API 호출 최적화 (여러 이벤트를 한 번에 분석)

**Step 4: A/B 테스트 (3주)**
- 기존 방식(키워드 only) vs LLM 보강 방식 병행 운영
- 지정학 리스크 인덱스 정확도 비교
- 섹터 영향 예측 정확도 비교 (사후 검증)

### 3.5 리스크 및 대응

| 리스크 | 대응 방안 |
|--------|---------|
| API 비용 초과 | 일일 예산 상한 설정 ($20/day), 배치 처리로 호출 수 최소화 |
| 환각(Hallucination) | Factual Grounding (뉴스 원문 참조), 키워드 결과와 교차 검증 |
| API 가용성 | fallback to 키워드 방식 (기존 로직 100% 유지) |
| 지연시간 | 비동기 처리, 캐시 적극 활용, 실시간이 아닌 배치 갱신 |
| 프롬프트 인젝션 | 뉴스 입력 사전 정제, 구조화된 출력 형식 강제 |

## 4. Phase 2b: 애널리스트 리포트 RAG (예상 3-4개월)

### 4.1 목표

증권사 애널리스트 리포트를 **벡터 DB에 축적**하여:
- 투자 의견, 목표가, 섹터 전망을 자동 추출 및 종합
- 종목별 컨센서스를 정량화하여 스코어링에 반영
- 캐피탈사 투자 의사결정의 참고 자료 자동 생성

### 4.2 아키텍처

```
[데이터 수집 계층]
리포트 수집 → PDF 파싱 → 텍스트 정제 → 메타데이터 추출
                                              │
                                              ▼
[벡터 저장 계층]
청킹(Chunking) → 임베딩(Embedding) → Vector DB 저장
    │                                        │
    ├── 의미 단위 청킹 (512-1024 토큰)       ├── 종목 코드 인덱싱
    └── 메타데이터 첨부 (날짜, 증권사, 종목)    └── 시간 기반 감쇠 가중치
                                              │
                                              ▼
[질의 처리 계층]
사용자 질의 → 의미 검색 → 관련 문서 추출 → LLM 종합 → 구조화된 결과
    │                                              │
    └── 자동 질의 (스코어링 시)                      └── 목표가, 투자의견, 리스크
```

### 4.3 기술 스택

| 구성요소 | 초기 선택 | 스케일 시 전환 | 사유 |
|---------|---------|--------------|------|
| PDF 파싱 | PyMuPDF + Unstructured | 동일 | 표/차트 추출 지원 |
| Embedding | text-embedding-3-small (OpenAI) | BGE-M3 (로컬) | 초기 비용 효율, 이후 비용 절감 |
| Vector DB | Chroma (로컬) | Qdrant (클러스터) | 초기 단순성, 이후 확장성 |
| Orchestration | LangChain | LlamaIndex | 프로토타이핑 속도 |
| LLM | GPT-4o | Claude / GPT-4o-mini | 종합 분석 품질 |

### 4.4 데이터 소스

**1차 소스 (즉시 활용 가능)**:
- 주요 증권사 리포트: KB증권, NH투자, 삼성증권, 미래에셋, 한국투자, 키움 등
- 공시 자료: DART (전자공시시스템) API
- 기업 IR 자료

**2차 소스 (단계적 추가)**:
- 뉴스 아카이브 (기존 수집 파이프라인 활용)
- 산업 분석 보고서 (한국은행, KDI, 산업연구원)
- 해외 리서치 (Bloomberg, Reuters)

### 4.5 핵심 활용

**스코어링 연동**:
- **목표가 컨센서스**: 증권사별 목표가를 수집하여 평균/중앙값/범위 표시
- **투자 의견 종합**: 매수/매도/중립 비율 정량화 → Fundamental 점수 보정 참고
- **섹터 전망 트렌드**: 업종 전망 변화를 시계열로 추적

**자동 분석 기능**:
- 종목 검색 시 최근 리포트 요약 자동 표시
- 리스크 요인 자동 추출 및 Risk 팩터 참고
- 이전 리포트 대비 의견 변화 추적 (상향/하향/유지)

### 4.6 비용 추정

| 항목 | 월 비용 (초기) | 월 비용 (안정화) | 비고 |
|------|-------------|---------------|------|
| Embedding API | ~$50-100 | ~$20-50 (로컬 전환) | 일 50-100건 리포트 기준 |
| LLM API (질의 처리) | ~$100-300 | ~$80-200 | 종목당 RAG 질의 1-2회 |
| Vector DB (Chroma/Qdrant) | ~$0 (로컬) | ~$50-150 (클라우드) | 데이터 증가에 따라 |
| PDF 처리 인프라 | ~$0 | ~$30-50 | 대량 처리 시 워커 추가 |
| **합계** | **~$150-400** | **~$180-450** | |

## 5. Phase 3: 자연어 인터페이스 (예상 4-6개월)

### 5.1 목표

- **"삼성전자 지금 살까?"** 같은 자연어 질의에 종합 답변 제공
- 6-팩터 스코어링 + 뉴스 감성 + 애널리스트 리포트 + 지정학 분석을 **종합한 자동 리포트 생성**
- 캐피탈사 투자 위원회에 제출 가능한 수준의 구조화된 분석 보고서

### 5.2 아키텍처

## 6. 기술 원칙 및 제약

### 6.1 절대 금지

다음 영역은 어떤 상황에서도 LLM으로 대체하지 않는다:

| 금지 영역 | 현재 구현 | 사유 |
|----------|---------|------|
| 기술적 분석 (MA, RSI, MACD, BB, OBV) | `backend/utils/technical.py` 규칙 기반 | 수학적 계산에 환각 개입 불가 허용 |
| 시그널 로직 (모멘텀, 평균 회귀, 브레이크아웃) | `backend/services/signal_service.py` 규칙 기반 | 트레이딩 로직의 결정론적 재현성 필수 |
| 리스크 계산 (변동성, MDD, VaR) | `backend/services/risk_service.py` 규칙 기반 | 리스크 수치의 정확성과 감사 가능성 보장 |
| 매크로 데이터 수집/계산 | `backend/services/macro_service.py` 규칙 기반 | 시장 데이터는 객관적 사실, LLM 해석 불필요 |
| 정량 지표의 LLM "직감" 대체 | 해당 없음 | 투자 의사결정의 근거 추적성 훼손 |

### 6.2 LLM 적합 영역

| 적합 영역 | Phase | 현재 상태 | LLM 역할 |
|----------|-------|---------|---------|
| 뉴스 감성 분석 | 1 (진행 중) | KR-FinBERT 앙상블 | 정성적 텍스트의 감성 이해 |
| 지정학 이벤트 심층 해석 | 2a | 키워드 기반 | 이벤트 맥락/방향성/연쇄 효과 분석 |
| 애널리스트 리포트 요약/종합 | 2b | 미구현 | 비정형 PDF의 핵심 정보 추출 |
| 자연어 질의 응답 | 3 | 미구현 | 다중 소스 통합 분석 및 보고서 생성 |
| 리포트 자동 생성 | 3 | 미구현 | 구조화된 투자 보고서 작성 |

### 6.3 환각(Hallucination) 대응 전략

**3중 검증 프레임워크**:

1. **Factual Grounding (사실 기반 고정)**
   - 모든 LLM 출력은 입력 데이터(뉴스 원문, 리포트 원문, 시장 데이터)로 교차 검증
   - 소스 문서 참조 의무화 (출처 없는 분석은 "미확인" 표시)

2. **정량 데이터 교차 검증**
   - LLM의 정성적 판단을 정량 지표(6-팩터 스코어)와 대조
   - 정성-정량 판단 불일치 시 경고 표시 및 로깅
   - 예: LLM이 "매수" 판단이나 Risk 등급 D/E → 불일치 경고

3. **Confidence Score 기반 필터링**
   - LLM 출력에 신뢰도 점수 부여 요청
   - 신뢰도 낮은 분석(<0.6)은 "확인 필요" 표시
   - 임계값 이하 결과는 사용자에게 추가 검토 권고
